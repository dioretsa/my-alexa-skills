My Weather skill

Overview

My Wether is an Amazon Alexa skill. It speaks weather observations and forecats for places in Finland. Observation and forecast data is based on the Finnish Meterological Institute (FMI) open data. FMI open data is licenced based on CC BY 4.0 by FMI.

Weather observations are read from a FMI weather station closest to the place given by the user. FMI has over 400 stations so the overall coverage is pretty good in most places in Finland. Weather forecast is based on RCR HIRLAM numerical forecast.

User can configure multiple different locations, i.e. 'places', and place names. User can then ask the skill for observations or forecasts for the given place name. 

Configuration

Places

User can add an address for the following place names:

home, work, school, gym, cabin, downtown, place1, place2 and place3

Places are added to Alexa's to-do list by using the following syntax:

place:<place_name>:[<street_address>],<city>

Notice that zip code is not needed. For example:

place:school:Nissintie 2,Espoo

You can also give less accurate addresses. For example:

place:school:Nissintie,Espoo

place:school:,Espoo

User can add multiple places into the to-do list. Finally, user then asks Alexa to save places:

"Alexa, ask My Weather to save places."

After places were successfully persisted then given to-do list items will be automatically removed by the skill. Adding a new place with the same place name will overwrite old data. Places configured by the user are kept in privacy and not used for any other purpose. For further information see the privacy policy description.  

If the user has not configured any places, then My Weather skill will retrieve observations and forecasts for the capital city Helsinki.

Permissions

Prior to using the My Weather skill, the user must grant list read and write permissions for the skill in the Alexa application.  

Example Utterances

  • "Alexa, open my weather."
    • This launches the interactive mode. Exit with exit, stop, good bye etc.
    • You can continue like "get observations" or "get forecast", like in one-shot commands listed below but without "Alexa, ask My Weather to..."
  • "Alexa, ask My Weather to get observations [for <place_name>]"
    • Speaks the latest weather observations for the given place. If the place is not specified then observations will be for home, provided that it has been configured (if not then Helsinki will be used).
    • For example: "Alexa, ask My Weather to get observations."
    • For example: "Alexa, ask My Weather to get observations for work."
  • "Alexa, ask My Weather to get forecast [for <place_name>] [at [tomorrow] <morning|noon|afternoon|evening|night>]"
    • Speaks the latest weather forecast for the given place and moment in today or tomorrow. If the place is not specified then forecast will be for home, provided that it has been configured (if not then Helsinki will be used). If the moment is not specified then the next available forecast will be used. Corresponding wall clock times for moments are (starting from morning): 6, 12, 15, 18 and 21.
    • For example: "Alexa, ask My Weather to get forecast."
    • For example: "Alexa, ask My Weather to get forecast at night."
    • For example: "Alexa, ask My Weather to get forecast for downtown."
    • For example: "Alexa, ask My Weather to get forecast for home at tomorrow afternoon."
